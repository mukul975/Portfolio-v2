How It Works

AI agents read only the YAML frontmatter (~40 tokens) to decide relevance — then load the full workflow only when needed.

SKILL.md 
---
name: performing-memory-forensics
description: Analyze memory dumps to extract
  processes, connections, and malware
  artifacts using Volatility3.
domain: cybersecurity
subdomain: digital-forensics
tags: [forensics, volatility3,
  memory-analysis, incident-response]
---

## When to Use
Trigger when investigating a compromise
and need to analyze volatile evidence...

## Prerequisites
- Volatility3 installed
- Memory dump acquired (dd/winpmem)
- Python 3.8+ environment

## Workflow
1. Identify OS profile
2. List running processes (windows.pslist)
3. Detect injected code (windows.malfind)
...

name: aws-s3-bucket-security-audit
description: Audit S3 bucket configurations
  for public exposure, encryption gaps,
  and misconfigured access policies.
domain: cybersecurity
subdomain: cloud-security
tags: [aws, s3, cloud-security,
  misconfiguration, data-exposure]
---

## When to Use
Use when assessing AWS environment for
publicly accessible or poorly secured
S3 buckets...

## Prerequisites
- AWS CLI configured
- IAM permissions: s3:List*, s3:Get*
- jq installed

## Workflow
1. List all buckets (aws s3 ls)
2. Check public access settings
3. Audit bucket policies and ACLs
...
---
name: kerberoasting-with-impacket
description: Extract Kerberos service tickets
  for offline cracking to identify weak
  service account passwords in AD.
domain: cybersecurity
subdomain: red-teaming
tags: [kerberos, active-directory,
  impacket, credential-access]
---

## When to Use
Use during internal pentest when you
have domain credentials and need to
escalate privileges...

## Prerequisites
- Impacket installed
- Valid domain credentials
- Network access to DC

## Workflow
1. Run GetUserSPNs.py to enumerate SPNs
2. Request service tickets (-request)
3. Crack hashes with hashcat -m 13100
...
